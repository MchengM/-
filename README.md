基于机器学习可解释性的信贷风险预测项目，针对信贷数据不均衡问题，通过多模型对比与 SHAP 可解释性分析，实现精准风险识别与业务可追溯，满足金融场景风控需求。
项目简介：本项目基于 Kaggle 信贷违约预测数据集（https://www.kaggle.com/competitions/credit-default/data），围绕 “数据处理 - 模型训练 - 可解释性分析” 全流程，构建适配信贷场景的风险预测体系。核心目标包括：解决信贷数据不均衡问题、对比 7 类机器学习模型性能、通过 SHAP 工具实现模型可解释性、确保项目工程化可复现。项目最终筛选出的 LightGBM 模型在测试集上实现 89% 准确率、87% F1 分数与 0.93 AUC 值，核心特征影响符合 “信用评分、收入能力、贷款金额为风险关键因素” 的业务逻辑。
技术栈：数据处理--Pandas、NumPy--数据加载、缺失值填补、特征筛选与清洗
       特征工程--Scikit-learn--离散特征编码、连续特征标准化、数据划分
       不均衡数据处理--imblearn（SMOTE）--训练集过采样，平衡正负样本分布
       机器学习模型--Scikit-learn、XGBoost、LightGBM--逻辑回归、随机森林、XGBoost、LightGBM 等模型训练
       参数优化--Scikit-learn（GridSearchCV）--5 折交叉验证 + 网格搜索，优化模型超参数
       模型评估--Scikit-learn（roc_auc_score/confusion_matrix）、Matplotlib--计算 AUC/F1 分数、绘制 ROC 曲线与混淆矩阵
       可解释性分析--SHAP--特征全局重要性分析、单样本预测贡献度解释
       版本控制--Git--代码提交与仓库管理
依赖环境  1. 操作系统 Windows 10/11、macOS 12+、Linux（Ubuntu 20.04+）
        2. Python 版本  Python 3.8 ~ 3.10（推荐 3.9，避免高版本兼容性问题）
        3. 第三方库安装 通过pip命令安装所有依赖，建议先创建虚拟环境
项目可复现性说明
1、数据集：原始数据来自 Kaggle 公开竞赛，可直接下载，预处理逻辑在代码中完全开源；
2、模型参数：所有模型的最优参数通过网格搜索自动输出，保存在results/model_performance.csv中；
3、实验环境：依赖库版本精确指定，避免因版本兼容导致结果差异；
4、运行流程：main.py一键复现全流程，无需手动干预，结果文件自动生成且路径固定。
